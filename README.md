# DeepSOC 智能安全运营中心系统

一个面向中国大学生计算机设计大赛人工智能应用/软件应用与开发赛道的智能安全运营中心系统。项目基于现有课程项目继续演进，利用本地大模型、向量检索、联网情报和前端可视化能力，构建面向网络安全事件的诊断、溯源、研判与响应闭环。

本项目不再将自己定义为通用故障诊断工具，而是聚焦企业安全运营中心场景，目标是帮助分析员完成安全日志理解、攻击链还原、威胁情报比对和防御策略生成等高价值任务。

---

## 📌 项目定位

在真实安全运营场景中，分析员经常面对以下问题：

- 日志来源复杂，包含主机日志、Web 访问日志、异常流量特征和人工整理报告
- 安全事件分析链条长，需要同时完成检索、比对、推理、研判和修复建议生成
- 数据往往带有内网 IP、业务字段和敏感配置，不能直接上传到公有云进行原始分析
- 传统聊天式界面难以充分展示攻击路径、调查时间线和处置建议，演示效果与工程感不足

基于这些问题，DeepSOC 将系统升级为端云结合、多智能体协同、可视化呈现的安全运营控制台，以更契合国赛答辩场景和安全类项目展示需求。

## 🌟 核心创新

### 1. 端云结合的混合模型架构

系统以本地模型保障数据隐私，以云端模型补足复杂推理能力，形成面向安全场景的智能路由机制。

| 模块 | 角色定位 | 主要任务 |
|------|----------|----------|
| Local 模型层 | 基于 Ollama 的本地轻量模型 | 日志脱敏、字段清洗、常见攻击特征初筛、基础问答 |
| Cloud 模型层 | 基于 API 的云端高性能模型 | 复杂攻击链推理、APT 溯源、多源证据综合研判 |
| Router 路由层 | 请求分发与策略判断 | 根据数据敏感度、任务复杂度和时延要求进行模型选择 |

当前课程项目已经具备 Ollama 本地模型调用能力，并包含 langchain-openai 相关依赖，为竞赛版本引入 DeepSeek API 提供了基础。竞赛版重点升级方向是将原有的“模型切换”能力进一步演进为“任务级智能分流”。

### 2. 面向安全研判的多智能体协同

为避免单一模型在复杂攻防任务中出现上下文遗忘和幻觉，竞赛版采用多智能体协同思路，将安全分析流程拆分为多个专业角色。

| Agent | 职责 | 可复用的现有能力 | 竞赛升级方向 |
|-------|------|------------------|--------------|
| Threat Intel Agent | 联网检索最新威胁情报、CVE 和安全社区信息 | DuckDuckGo 联网搜索 | 引入结构化情报摘要与漏洞标签归档 |
| Forensic Agent | 在本地 RAG 库中检索相似案例，分析攻击入口与关键证据 | 向量检索、日志处理、历史会话上下文 | 强化 IOC 提取、时间线重建和攻击路径还原 |
| Remediation Agent | 生成修复建议、防御脚本和策略模板 | 代码生成、Markdown 渲染、代码高亮 | 输出 WAF 规则、Nginx 拦截脚本、iptables 模板 |
| Orchestrator Agent | 面向用户交互，编排其他 Agent，汇总结论 | 当前聊天主流程、上下文管理 | 基于 LangGraph 的流程编排与多阶段状态管理 |

当前仓库已经具备多轮对话、工具调用和检索增强的实现基础。竞赛版将以此为底座，进一步引入 LangGraph 或等价工作流编排机制，使系统从“单助手”升级为“安全专家团队”。

### 3. 从聊天窗口升级为 SOC 控制台

竞赛展示中，前端不再只是对话界面，而是具备专业感和指挥感的安全运营中心控制台：

- 左侧区域负责会话管理、模型切换、诊断任务入口
- 中央区域展示流式对话、多智能体推理过程和最终研判结论
- 右侧区域作为动态情报面板，展示漏洞命中结果、网络节点状态和处置建议摘要

这种改造能够让评委直观看到系统不仅能“回答”，还能“分析、追踪、汇总并辅助响应”。

### 4. 一图化展示攻击链与响应闭环

相较于纯文本回复，竞赛版更强调结构化展示：

- 将模型思考过程包装为自动化溯源时间线
- 将攻击路径渲染为拓扑图或 Kill Chain 图谱
- 将修复建议包装为可操作的策略卡片和伪执行面板
- 将安全告警标签渲染为高亮告警卡而非普通文本

## 🛡️ 系统能力设计

围绕安全运营完整流程，系统能力分为四个层级。

### 1. 检测

- 识别异常日志、异常流量特征和潜在攻击入口
- 支持文本、文档、表格等多源输入，后续扩展到安全日志导出文件和流量特征文件
- 对上传内容进行预处理、标准化和关键字段提取

### 2. 分析

- 结合本地知识库和联网情报进行证据增强
- 通过本地模型完成脱敏与基础模式识别
- 通过云端大模型完成复杂跨日志关联分析与深度推理

### 3. 溯源

- 检索历史相似攻击案例和处置记录
- 还原攻击者从外网入口到内部横向移动的链路
- 输出包含时间顺序、关键节点和证据来源的调查结论

### 4. 响应

- 自动生成防御建议、拦截规则和修复思路
- 以卡片化形式展示处置动作、适用条件和风险提示
- 为答辩演示提供“生成补丁文件”“一键下发防御策略”等工程化交互入口

为避免将未来规划写成既成事实，README 采用“当前基础 + 竞赛升级”的表述方式。

| 模块 | 当前基础 | 竞赛升级方向 |
|------|----------|--------------|
| 模型层 | 已支持本地模型调用和多模型切换 | 升级为本地脱敏 + 云端深推理的智能路由 |
| 检索层 | 已具备 ChromaDB 向量检索和关键词增强 | 强化安全场景知识库、漏洞情报和历史攻击案例库 |
| 工作流层 | 已具备工具调用和上下文传递能力 | 引入多智能体协同编排与状态追踪 |
| 展示层 | 已支持 Markdown 渲染、代码高亮和会话管理 | 改造为 SOC 仪表盘、时间线、情报面板和攻击拓扑图 |

### 3. 端云混合调用策略

竞赛版的模型调用策略分为三个阶段：

1. 本地预处理阶段：对原始日志完成脱敏、字段切分、噪声剔除和基础特征匹配。
2. 智能路由阶段：判断问题是否涉及高复杂推理、多源证据整合或高级威胁分析。
3. 云端深度分析阶段：仅将脱敏后的关键日志片段和提炼后的上下文发送到云端大模型完成深度研判。

这一设计既突出了数据安全意识，也能体现项目在工程落地上的合理性。

## 🤖 多智能体协同方案

### Threat Intel Agent

- 调用联网搜索能力，获取最新 CVE、公开攻击样本和安全社区情报
- 将原始搜索结果整理为结构化情报摘要
- 为 Forensic Agent 提供漏洞背景与已知攻击手法参考

### Forensic Agent

- 从本地向量库和历史案例库中检索相似事件
- 提取 IOC、可疑路径、异常 IP、关键时间点和攻击入口
- 输出可用于可视化的事件链结构数据

### Remediation Agent

- 根据前两个 Agent 的结论生成防护建议
- 输出适配 Web、主机和网络边界场景的策略模板
- 将代码高亮区转化为更具工程感的处置卡片和脚本面板

### Orchestrator Agent

- 接收用户指令并拆分子任务
- 决定是否需要启用联网搜索、本地检索或深度推理
- 汇总各 Agent 结果并输出最终的诊断、溯源和修复闭环报告

### 预期输出格式

为便于前端渲染，竞赛版建议统一多智能体输出结构，包括：

- 风险等级
- 攻击类型标签
- 关键证据列表
- 攻击路径节点与边
- 修复建议与执行优先级
- 可选的防御脚本片段

## 💻 Vue 3 深度改造方案

### 1. 视觉风格与布局重塑

前端整体风格改造为暗黑科技风的安全战术面板：

- 深色背景搭配荧光绿、战术蓝和告警红，强化安全场景识别度
- 基于全局 CSS 变量设计主题体系，支持常规模式和 SOC 模式切换
- 从原有左右布局升级为三栏布局或仪表盘布局

建议布局如下：

| 区域 | 主要内容 | 展示目标 |
|------|----------|----------|
| 左栏 | 会话列表、模型选择、快捷诊断入口 | 提升操作效率 |
| 中栏 | 主对话区、推理过程、流式输出 | 展示智能分析过程 |
| 右栏 | 动态情报面板、漏洞命中、节点状态 | 强化工程演示效果 |

### 2. ChatMessage.vue 的升级方向

现有系统已经支持思维链和 Markdown 渲染，竞赛版将其强化为自动化溯源时间线：

- 将“思考中”改造成多智能体执行状态播报
- 按步骤展示侦查、比对、研判、汇总过程
- 对告警标签进行专用渲染，例如将 `[Alert: SQL Injection]` 渲染为告警卡片
- 保留 highlight.js 的基础上，增加安全规则片段的高亮与风险等级提示

### 3. 攻击链路拓扑图

为了强化可视化表达，竞赛版计划引入 ECharts 或 G6：

- 当前端接收到攻击路径 JSON 时，自动渲染为攻击链路拓扑图
- 展示从外部 IP、入口服务、权限提升到内网横向移动的关键过程
- 支持节点状态、风险等级和证据来源的可视化映射

### 4. 修复代码与执行面板强化

现有系统已经具备代码块高亮、复制和在线运行基础，竞赛版将交互语义升级为：

- 一键生成防御策略
- 导出补丁建议
- 伪执行响应命令
- 标注预期效果、适用环境和回滚提示

这类设计能在不真正执行危险操作的前提下，展示出系统具备“辅助响应”的工程感。

### 5. ChatInput.vue 的安全场景扩展

在当前支持 `.txt`、`.docx`、`.xlsx` 的基础上，竞赛版重点扩展安全输入体验：

- 为安全日志、流量特征文件和导出报告设计更专业的上传提示
- 上传时加入“提取安全特征词”的过程动画
- 在输入框上方加入快捷指令盘，例如：
  - 一键检测 WebShell
  - 分析最新异常流量
  - 全盘漏洞扫描评估

## 🎯 典型演示流程

为了方便比赛演示和答辩说明，推荐使用以下端到端场景：

1. 用户上传一份包含异常访问记录的安全日志或流量特征文件。
2. 本地模型先完成脱敏、清洗和初步攻击特征识别。
3. Forensic Agent 在本地知识库中检索相似历史案例并提取关键证据。
4. Threat Intel Agent 联网比对最新漏洞和攻击样本。
5. Orchestrator Agent 汇总多个 Agent 结果，生成调查时间线和攻击链路。
6. 右侧情报面板展示漏洞命中、风险等级和可疑节点状态。
7. Remediation Agent 生成 WAF 规则、Nginx 拦截配置或处置建议卡片。
8. 前端以拓扑图、时间线和策略面板三种形式完成可视化展示。

## 🚀 快速开始

### 1. 后端部署

```bash
git clone https://github.com/EverVio/DeepSOC.git
cd django_backend
pip install -r requirements.txt
python manage.py runserver
```

### 2. 前端部署

```bash
cd vue_frontend
npm install
npm run dev
```

### 3. 本地模型准备

```bash
ollama pull deepseek-r1:7b
ollama pull bge-large:latest
```

### 4. 云端模型配置

竞赛版建议增加云端 API 配置项，用于接入 DeepSeek-V3 或 DeepSeek-R1 等高性能模型。推荐通过环境变量方式管理密钥，例如：

```bash
set DEEPSEEK_API_KEY=your_api_key
```

### 5. 体验方式

1. 打开前端页面 http://localhost:8092。
2. 使用默认账号登录，用户名可自定义，密码为 `secret`。
3. 上传日志或输入安全分析任务。
4. 观察系统在对话区、情报面板和可视化区域中的输出。
5. 根据需要切换模型、会话和检索策略。

## 🏆 参赛价值总结

DeepSOC 的核心价值不在于简单叠加大模型能力，而在于将课程项目升级为一个更接近真实安全运营中心工作流的智能系统：

- 在架构上，强调数据隐私与复杂推理兼顾的端云结合设计
- 在智能上，强调多智能体分工协作而非单体回答
- 在交互上，强调专业化、可视化和工程感并重的 SOC 控制台体验
- 在展示上，能够形成“诊断 - 溯源 - 修复”的完整闭环，适合比赛演示和答辩说明

如果后续继续完善模型路由、Agent 编排和可视化模块，本项目能够从课程作业进一步演进为具有较强竞赛辨识度和落地潜力的人工智能安全应用作品。

## 🗂️ 未实现功能建议实现顺序

前端 SOC 控制台的主体视觉框架（三栏布局、战术分析终端、动态图表区域）已经基本搭建完成，当前的开发策略应从“先底座后展示”调整为“协议先行，数据打通，逐步替换底层引擎”。建议按照以下顺序实施：

第一阶段：前后端协议拉通与视觉语义化（最高优先级）
当前首要任务是让现有的高颜值前端 UI 展示真实的“安全业务”，而非通用 IT 故障或随机动画。

统一输出 Schema 与前后端协议
在后端预定义一套固定的 JSON Schema（包括节点、边、风险等级、CVE编号、IOC列表、修复脚本等）。目前前端的图表需要真实数据驱动，定义好契约是后续所有开发的基础。

拓扑图与情报面板的静态数据绑定
改造前端组件，使其能够消费上述 Schema。将 GLOBAL ATTACK TOPOLOGY 的随机粒子动画替换为基于 ECharts 或 G6 的真实攻击路径渲染（Kill Chain）；将右侧的常规占比图替换为漏洞命中列表或可交互的响应卡片。此时可先用一套硬编码的“高分剧本”（如 WebShell 注入到内网横向移动）跑通前后端交互链路，确保界面演示效果无懈可击。

第二阶段：核心安全数据底座建设
在前端演示效果有保障的前提下，开始夯实后端实际的数据处理能力。

安全数据集收集与向量检索底座增强
搜集并整理安全案例库、CVE/漏洞情报归档、常见 Web 攻击模式和处置模板库，完成清洗、切分与向量入库。这是让系统具备真实诊断能力的核心“弹药库”。

安全数据预处理与统一结构化层
补齐日志脱敏、字段清洗、日志类型识别和 IOC（如恶意 IP、Hash）提取。将上传的无格式日志转化为系统可理解的标准化安全事件对象，为大模型推理提供干净的上下文。

第三阶段：多智能体协作与动态路由
利用大模型能力替换第一阶段的“硬编码剧本”，实现真正的智能分析。

端云结合的模型路由机制
在现有系统上实现任务级路由：本地轻量模型（如 Ollama）负责日志脱敏、初筛和字段提取，云端高性能模型（如 DeepSeek API）负责复杂攻击链推理。

垂直安全 Agent 落地

Forensic Agent（取证）： 基于日志和向量库，提取攻击入口和关键时间点，输出攻击链路节点数据（供拓扑图渲染）。

Threat Intel Agent（情报）： 结合联网搜索和漏洞库，输出结构化的威胁情报摘要（供右侧情报面板渲染）。

Remediation Agent（响应）： 输出 WAF 规则或防御脚本（供前端伪执行面板展示）。

Orchestrator Agent 与工作流编排
引入 LangGraph 或等价状态机机制，统一调度上述三大 Agent。接收用户的自然语言指令或日志文件，规划分析路径，并汇总多智能体的结果，严格按照第一阶段定义的 Schema 格式返回给前端。

第四阶段：赛前冲刺与体验打磨
安全交互增强与快捷操作入口
在现有输入框周围增加工程化交互：如“安全日志拖拽上传解析动画”、“一键检测 WebShell 快捷指令”、“点击拓扑图节点自动下发隔离策略（伪执行）”。这些小功能开发成本低，但能大幅拉升控制台的“实战感”。

竞赛演示场景固化与评测指标补齐
锁定 1-2 个最具代表性的安全攻防剧本用于终审答辩。固化对应的日志样本、预置的检索库状态，确保现场演示 100% 稳定，并整理系统的响应时延、命中准确率等量化指标写入答辩 PPT。
